## 📘ส่วนที่ 1: JSX Syntax & JSX Rules

### 📄คำถาม 1.1: จงอธิบายความแตกต่างระหว่าง class และ className ใน JSX พร้อมทำให้ตัวอย่างที่แสดงเหตุผลว่าทำไม React ต้องการใช้ className

```text
JSX ต้องใช้ className แทน class เพราะ:
1. class เป็นคำสงวน (reserved keyword) ใน JavaScript
2. JSX แปลง HTML attribute ให้เป็น JavaScript properties
3. ตัวอย่าง:
   - ถูก: <div className="btn">...</div>
   - ผิด: <div class="btn">...</div> (error)

ใน JavaScript ทั่วไป, className ใช้เพื่อหลีกเลี่ยงการชนกับคำสงวน class

```

### 📄คำถาม 1.2: จงเขียน Component ที่ใช้ JSX rules ให้ถูกต้องตามตัวอย่าง

```jsx
// ต้องเขียนให้ถูกต้อง (แก้ไขโค้ดที่ผิด)

```

## 📘ส่วนที่ 3: Props

### 📄คำถาม 3.1: จงสร้าง Component ชื่อ ProductCard ที่รับ Props: name, price, inStock พร้อมอธิบายวิธีการส่ง Props จาก Parent Component

```jsx
// ต้องสร้าง Component นี้:
// 1. ProductCard Component
// 2. แสดงวิธีการใช้ใน App.jsx (ส่ง Props)
// 3. อธิบายความแตกต่างระหว่าง string, number, boolean props
//   - string: ส่งค่าเป็นข้อความ
//   - number: ส่งค่าเป็นตัวเลข
//   - boolean: ส่งค่าเป็น true/false

// Component ProductCard
function ProductCard({ name, price, inStock }) {
  const nameProduct = name;
  const priceProduct = price;
  const isStock = inStock;

  return (
    <div>
      <h2>🍫 {nameProduct}</h2>
      <h3>฿ {priceProduct}</h3>
      <p>สถานะ: {isStock ? "มีสินค้า" : "หมด"}</p>
    </div>
  );
}

// Main Component
export default function App() {
  return (
    <>
      <ProductCard name={"Chocolate"} price={99} inStock={true} />
    </>
  );
}
```

### 📄คำถาม 3.2: Props destructuring คืออะไร? จงแสดงตัวอย่างเปรียบเทียบระหว่าง:

- วิธีเก่า: function Card(props) { ... }
- วิธีใหม่: function Card({ title, description }) { ... }
- และอธิบายว่าทำไมวิธีใหม่ (destructuring) จึงเป็น best practice

```text
วิธีเก่า: รับ Props ทั้งก้อน (Without Destructuring) วิธีนี้เราจะรับ parameter ตัวเดียวชื่อ props แล้วเวลาจะใช้งานข้อมูลข้างใน ต้องอ้างอิงผ่าน props.ชื่อตัวแปร เสมอ

วิธีใหม่: ระบุชื่อตัวแปรที่ต้องการไว้ในเครื่องหมายปีกกา { } ตรงช่อง parameter ของฟังก์ชันได้เลย React จะจัดการดึงค่าจาก Object props มาใส่ตัวแปรเหล่านั้นให้โดยอัตโนมัติ

ทำไมการทำ Destructuring จึงเป็น Best Practice?
ความชัดเจน (Readability & Intent): เพียงแค่ดูที่บรรทัดแรกของ Component (Function Signature) เราจะรู้ทันทีว่า Component นี้ต้องการข้อมูลอะไรบ้างเพื่อทำงาน โดยไม่ต้องไปไล่อ่านโค้ดข้างล่างว่ามี props.something ตรงไหนบ้าง
```

## 📘ส่วนที่ 4: Lists & .map()

### 📄คำถาม 4.1: จงอธิบายว่า .map() ทำงานอย่างไรในการแสดงรายการข้อมูล พร้อมตัวอย่างโค้ดที่แสดง:

- การใช้ .map() เพื่อแสดง list ของ Component
- เหตุผลว่าทำไมต้องใช้ key property
- ข้อแตกต่างระหว่าง key={index} และ key={item.id}

```jsx
// React ใช้ key เป็นเสมือน "ป้ายชื่อ" เพื่อติดตามว่า Element ตัวไหนในรายการมีการ เปลี่ยนแปลง, เพิ่มเข้า หรือถูกลบออก
// key={index} (ลำดับของ Array) อันตราย, key={item.id} (ID จาก Database) ปลอดภัย

// Component ProductList
function ProductList({ name, price }) {
  return (
    <div>
      <h1>{name}</h1>
      <h2>{price}</h2>
    </div>
  );
}

// Main Component
export default function App() {
  const cars = [
    { id: 1, name: "Tesla", price: "1,149,000" },
    { id: 2, name: "BMW", price: "2,899,000" },
    { id: 3, name: "Lamborghini", price: "8,750,000" },
  ];

  return (
    <div>
      {cars.map((car) => (
        <ProductList key={car.id} name={car.name} price={car.price} />
      ))}
    </div>
  );
}
```

### 📄คำถาม 4.2: สร้าง Component ที่ใช้ .map() เพื่อแสดง:

- รายชื่อนิสิต (array of objects ที่มี id, name, grade)
- แสดงเป็นตาราง โดยใช้ .map() ในการสร้าง แต่ละแถว
- อธิบายเหตุผลว่าทำไมต้องใช้ unique ID เป็น key

```jsx
// ใน React การกำหนด key={student.id} (ซึ่งเป็นค่าที่ไม่ซ้ำกัน) มีความสำคัญอย่างยิ่ง คือ ระบุตัวตนที่แม่นยำ (Identity) เพื่อช่วยให้ทราบว่าไอเทมไหนในรายการคือตัวไหนกันแน่ หากมีการเปลี่ยนแปลงข้อมูลจะใช้ ID นี้ในการอ้างอิงเพื่ออัปเดตเฉพาะจุดที่เปลี่ยน แทนที่จะต้องลบตารางทิ้งแล้วสร้างใหม่ทั้งหมด

// Component StudentTable
function StudentTable() {
  const students = [
    { id: 1, name: "Freddy", grade: "A" },
    { id: 2, name: "Bonnie", grade: "B" },
    { id: 3, name: "Chica", grade: "C" },
    { id: 4, name: "Foxy", grade: "D" },
  ];

  return (
    <div>
      <h2>รายชื่อนิสิต</h2>
      <table
        border="1"
        style={{
          width: "100%",
          textAlign: "center",
          borderCollapse: "collapse",
        }}
      >
        <thead>
          <tr>
            <th style={{ padding: "10px" }}>รหัสนิสิต</th>
            <th style={{ padding: "10px" }}>ชื่อ</th>
            <th style={{ padding: "10px" }}>เกรด</th>
          </tr>
        </thead>
        <tbody>
          {students.map((student) => (
            <tr key={student.id}>
              <th style={{ padding: "10px" }}>{student.id}</th>
              <th style={{ padding: "10px" }}>{student.name}</th>
              <th style={{ padding: "10px" }}>{student.grade}</th>
            </tr>
          ))}
        </tbody>
      </table>
    </div>
  );
}

// Main Component
export default function App() {
  return <StudentTable />;
}
```

## 📘ส่วนที่ 5: Best Practices & Naming Conventions

### 📄คำถาม 5.1: จงอธิบายและยกตัวอย่างการตั้งชื่อในแต่ละประเภท ทั้งแบบที่ถูก และแบบที่ผิด:

- Component Names - ตั้งชื่อแบบไหน?
- Variable Names - ตั้งชื่อแบบไหน?
- Constants - ตั้งชื่อแบบไหน?
- File Names - ตั้งชื่อแบบไหน?

```text
Component Names - ตั้งชื่อแบบไหน?
ต้องเริ่มด้วยตัวพิมพ์ใหญ่ (PascalCase)
✅: UserList
❌: userlist

Variable Names - ตั้งชื่อแบบไหน?
ควรใช้เป็น camelCase
✅: userName
❌: user_name

Constants - ตั้งชื่อแบบไหน?
ควรใช้เป็น UPPER_SNAKE_CASE:
✅: MAX_VALUE
❌: maxValue

File Names - ตั้งชื่อแบบไหน?
ตั้งชื่อตาม Component (PascalCase):
✅: UserProfile
❌: userProfile
```

### 📄คำถาม 5.2: จากโค้ด Component ต่อไปนี้ให้บอก:

- ส่วนที่ตั้งชื่อถูกต้อง
- ส่วนที่ตั้งชื่อผิด (ถ้ามี)
- แก้ไขโค้ดให้เป็น best practice

```jsx
// Components ควรตั้งชื่อตัวแรกเป็นตัวอักษรพิมพ์ใหญ่ จาก user_profile เป็น UserProfile
// การกำหนดตัวแปรไม่ควรมีอักษรพิเศษเช่น _
// ชื่อตัวแปรควรตั้งชื่อให่สื่อความหมายใช้ตัวพิมพ์คั่นระหว่างคำ
function UserProfile({ userName, userAge, isPremium }) {
  const userData = {
    name: userName,
    age: userAge,
    isPremium: isPremium,
  };

  return (
    <div>
      <h3>{userData.name}</h3>
      <p>{userData.age} ปี</p>
    </div>
  );
}
```
